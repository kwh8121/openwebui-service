# lib

## Purpose
The shared frontend library, imported anywhere in the app as `$lib/...`. Everything reusable across routes lives here: typed API clients (`apis/`), Svelte components (`components/`), global stores (`stores/`), i18next translation resources (`i18n/`), Web Worker entry points (`workers/`), Pyodide glue (`pyodide/`), shared utilities (`utils/`), TypeScript types (`types/`), and app-wide constants. The `constants.ts` / `shortcuts.ts` / `dayjs.js` files at this level are legacy-flat singletons that predate the `constants/` subdirectory.

## Key Files

| File | Description |
|------|-------------|
| `constants.ts` | App-wide constant values imported by many routes (URLs, defaults) |
| `shortcuts.ts` | Keyboard shortcut definitions and dispatcher |
| `dayjs.js` | Preconfigured dayjs instance with plugins and locale wiring |
| `emoji-groups.json` | Emoji category metadata for the picker (~116 KB) |
| `emoji-shortcodes.json` | `:shortcode:` → emoji map (~52 KB) |
| `index.ts` | Small barrel export |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `apis/` | Typed HTTP clients for backend routes — one folder per feature (`chats/`, `auths/`, `ollama/`, `openai/`, `retrieval/`, …). Mirrors `../../backend/open_webui/routers/`. |
| `components/` | Svelte UI components grouped by feature (`admin/`, `chat/`, `workspace/`, `channel/`, `notes/`, `calendar/`, `automations/`, `common/`, `icons/`, `layout/`, `playground/`, `app/`) |
| `stores/` | Svelte stores holding shared reactive state |
| `utils/` | Non-UI helper functions (`characters/`, `marked/`, `transitions/`, and top-level `.ts` helpers) |
| `i18n/` | i18next resources; contents are (re)generated by `npm run i18n:parse` — see below |
| `i18n/locales/` | One folder per locale with translation JSON |
| `workers/` | Web Worker entry points bundled by Vite |
| `pyodide/` | Runtime glue that loads Pyodide assets fetched by `scripts/prepare-pyodide.js` |
| `types/` | Shared TypeScript type declarations |
| `constants/` | Newer constants split by domain (prefer this over `constants.ts` for new values) |

## For AI Agents

### Working In This Directory
- Import as `$lib/...`, never as relative paths that cross into `../routes/`.
- Translatable strings must go through i18next helpers (`$t(...)` / `i18n.t(...)`). Run `npm run i18n:parse` after adding new keys and commit the regenerated `i18n/` output — CI enforces a clean tree.
- Add new API clients under `apis/<feature>/index.ts` and mirror the backend router name; keep the return-type shape aligned with the FastAPI response models.
- Prefer adding new constants under `constants/<domain>.ts` rather than growing the flat `constants.ts`.
- `pyodide/` depends on assets that `prepare-pyodide.js` writes into `../../static/pyodide/`; do not check those in.

### Testing Requirements
- Type check: `npm run check` from repo root.
- Unit tests: `npm run test:frontend -- src/lib/<path>` (Vitest, `--passWithNoTests` by default).
- Format / lint: `npm run format` and `npm run lint:frontend` (both mutating).

### Common Patterns
- Components are `.svelte` files with `<script lang="ts">`; Svelte 5 runes (`$state`, `$derived`, `$effect`) coexist with classic reactive `$:` blocks — match the file you are editing.
- API client functions typically take a `token: string` first argument and throw on non-2xx.
- Stores are plain Svelte `writable`/`derived`; avoid custom store abstractions unless the pattern already exists nearby.

## Dependencies

### Internal
- Consumed by everything under `../routes/`.
- API clients in `apis/` shadow the backend routers at `../../backend/open_webui/routers/`; keep names aligned when adding new routes.

### External
- `svelte@5`, `i18next` (+ browser detector + resources-to-backend), `socket.io-client`, `dayjs`, `marked`, `dompurify`, `codemirror`, `@tiptap/*`, `prosemirror-*`, `chart.js`, `mermaid`, `pyodide`, `pdfjs-dist`, `xlsx`, `@huggingface/transformers` — full list in `../../package.json`.
