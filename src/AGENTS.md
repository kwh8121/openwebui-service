# src

## Purpose
SvelteKit 2 + Svelte 5 frontend source. `routes/` is the file-based router (SvelteKit conventions: `+page.svelte`, `+layout.svelte`, `+error.svelte`) and `lib/` is the shared library imported as `$lib/*`. The built output is written to `../build/` by `npm run build` and packaged into the Python wheel as `open_webui/frontend` — do not edit built assets.

## Key Files

| File | Description |
|------|-------------|
| `app.html` | SvelteKit HTML shell; injects `%sveltekit.head%` and `%sveltekit.body%` |
| `app.css` | Global stylesheet loaded by the root layout |
| `tailwind.css` | Tailwind CSS entry (v4); imported by `app.css` |
| `app.d.ts` | SvelteKit ambient type declarations (`App.Locals`, `App.PageData`, etc.) |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `lib/` | Shared code imported as `$lib/*` — API clients, components, stores, i18n, utilities (see `lib/AGENTS.md`) |
| `routes/` | SvelteKit file-based router — pages, layouts, endpoints (see `routes/AGENTS.md`) |

## For AI Agents

### Working In This Directory
- Import shared code as `$lib/...` (configured in `../svelte.config.js`), not relative paths.
- Do not edit `../build/`, `../static/pyodide/`, or `../static/sql.js/` — they are generated by `prepare-pyodide.js` or Vite.
- Translatable strings must be added via i18next helpers so `npm run i18n:parse` picks them up; commit the regenerated `lib/i18n/` files.

### Testing Requirements
- Type check: `npm run check` (runs `svelte-kit sync` + `svelte-check`).
- Unit tests: `npm run test:frontend -- <path>` (Vitest).
- Lint: `npm run lint:frontend` (mutating; use `npx prettier --check` for a read-only pass).

### Common Patterns
- Svelte 5 runes syntax is available (`$state`, `$derived`, `$effect`) — the codebase mixes classic reactive statements with runes; match the surrounding file's style.
- SvelteKit adapter is `@sveltejs/adapter-static` for the wheel build, per `svelte.config.js`.

## Dependencies

### Internal
- Consumed by `../svelte.config.js` and `../vite.config.ts`.
- Built output feeds `../backend/open_webui/frontend` at packaging time (see `../pyproject.toml` `force-include`).

### External
- `@sveltejs/kit`, `svelte@5`, `vite@5`, `tailwindcss@4`, `i18next`, `socket.io-client` — see `../package.json`.
