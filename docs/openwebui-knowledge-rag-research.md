# Open WebUI: 지식베이스와 RAG

## 개념과 적합한 자료

Knowledge는 PDF, 스프레드시트, 코드, 텍스트 문서를 업로드해 모델이 필요 시 검색·열람하도록 하는 RAG 기반 저장소다. 대량 기사 아카이브, 스타일북, 정책 문서, 공공 데이터 설명서처럼 전체를 매번 대화에 넣기 어려운 자료에 적합하다. 짧고 항상 필요한 편집 규칙은 Note 또는 Full Context 첨부가 더 정확할 수 있다.

## 검색 방식

- Focused Retrieval: 기본값이다. 의미 기반 벡터 검색에 BM25 키워드 검색과 재순위화를 결합할 수 있어 대형 컬렉션에 적합하다.
- Full Context: 문서 전체를 매 메시지에 넣는다. 짧은 스타일 가이드나 고정된 기사 템플릿에 유리하지만 컨텍스트 창을 빠르게 소진한다.
- Native function calling: 모델이 지식베이스를 목록화, 의미 검색, 정확 문자열/정규식 검색, 파일 열람하며 능동적으로 조사한다. 이 모드에서는 첨부 KB가 자동 주입되지 않으므로, 모델 프롬프트에 KB 도구 사용을 지시하거나 Full Context를 선택해야 한다.

## 편집국 권장 구조

- `Stylebook`: 사내 영문 스타일, 표기, 헤드라인, 윤리 기준. 전용 "Copy Editor" 모델에 Full Context 또는 소규모 KB로 연결한다.
- `Archive`: 발행 기사와 정정 기록. 섹션·연도·매체별 디렉터리와 메타데이터를 유지하고, 기사 작성 모델에는 해당 비트만 연결한다.
- `Research`: 공공기관 보고서, 통계, 법령, 검증된 원문. 원문 URL, 발행일, 획득일, 라이선스를 파일 메타데이터나 파일명 규칙으로 남긴다.
- `Confidential`: 제보·엠바고·취재원 자료. Private/Restricted로 생성하고 최소 그룹에만 읽기 권한을 부여한다. 일반 모델에 전역 연결하지 않는다.

## 운영 절차

1. KB를 업무 단위와 기밀 등급별로 생성하고, 설명에 소유자·갱신주기·허용 용도를 기록한다.
2. 신뢰 가능한 원문만 넣고, OCR·추출 품질이 인용 정확도에 영향을 준다는 점을 표본 검사한다.
3. 모델에 필요한 KB만 붙여 검색 범위를 제한한다. KB를 붙이지 않은 모델은 사용자가 읽을 수 있는 KB 전체를 발견할 수 있다.
4. 답변에는 파일명, 페이지/문단, 원문 링크를 제시하도록 모델 지침을 둔다. 검색 결과는 근거 후보이지 자동 인용이 아니다.
5. 대규모 또는 주기적 동기화는 공식 `oikb`로 Git, Confluence, SharePoint, S3 등의 원본과 동기화한다. 업로드 후에는 처리 상태가 완료될 때까지 검색 결과에 의존하지 않는다.

## 권한과 한계

모델과 KB의 접근은 사용자·그룹 권한을 따른다. Private 또는 Restricted 리소스에 읽기/쓰기 권한을 명시적으로 부여한다. 다만 권한이 있다고 해서 보도 사용이 허용되는 것은 아니므로 저작권, 라이선스, 엠바고, 개인정보 정책을 별도 적용한다.
