# Open WebUI: 채팅, 조사, 멀티모달

## 제공 기능

하나의 대화에서 Ollama, OpenAI 호환 API, Anthropic 등 연결된 모델을 선택하고, 필요하면 모델을 바꾸거나 복수 모델의 답변을 비교할 수 있다. 파일·이미지 첨부, 웹 검색과 URL 읽기, 코드 실행, 음성 입출력, 이미지 생성/편집, 개인 메모리, 대화 폴더·태그·고정, 메시지 큐를 지원한다.

## 기본 사용 흐름

1. 업무 목적에 맞는 승인 모델 또는 전용 모델 프리셋을 선택한다.
2. 기사 브리프, 인터뷰 녹취, 문서 또는 URL을 첨부한다. 큰 자료 집합은 채팅 첨부보다 Knowledge Base를 사용한다.
3. 웹 조사에는 검색 결과의 출처 링크와 인용을 답변에 명시하도록 요청한다.
4. 초안, 제목 후보, 요약, 번역은 원문과 검증 대상 사실을 분리해 요청한다.
5. 결과를 노트나 채널로 옮겨 편집자가 검토하고, 사실·인용·저작권을 확인한 뒤에만 발행 시스템으로 전달한다.

## 편집국 활용 예시

- 속보 브리프: 여러 모델로 핵심 사실, 확인 필요 사항, 독자 영향, 후속 취재 질문을 병렬 생성한다.
- 인터뷰: 녹취를 올려 화자별 요약, 인용 후보, 주장별 근거 확인 목록을 만든다. 자동 전사는 원음과 반드시 대조한다.
- 해외 기사: 번역 초안과 함께 고유명사, 수치, 법률 용어, 직접 인용문을 원문 기준으로 재검증한다.
- 비주얼 기획: 이미지 생성은 콘셉트 시안으로만 사용하고, 생성물 표기와 라이선스 정책을 별도로 적용한다.

## 데이터와 품질 주의점

- 웹 검색은 최신성에는 유리하지만 검색 결과나 모델 요약이 사실이라는 보증은 아니다. 원 출처를 열어 날짜, 저자, 맥락, 독립 출처 교차 확인을 한다.
- 메모리는 대화 간 개인 사실을 유지한다. 취재원 정보, 비공개 제보, 미공개 원고가 저장되거나 다른 문맥에 노출되지 않도록 조직 정책으로 사용 범위를 정한다.
- 임시 대화와 대화 이력, 공유 범위를 업무 등급별로 정한다. 민감 취재는 승인된 모델·지식베이스·채널에서만 처리한다.
- 브라우저 내 Python 실행과 Open Terminal은 코드 및 파일 접근 위험이 있다. 기자용 기본 권한으로 활성화하지 않는다.
