# Open WebUI 기반 영자 언론사 편집국 청사진

## 목표와 원칙

Open WebUI를 기사 작성, 조사, 교열, 번역, 협업을 지원하는 내부 AI 작업대(workbench)로 사용한다. 이는 CMS나 편집 책임자를 대체하는 시스템이 아니다. 최종 취재 판단, 사실 확인, 법률·윤리 검토, 발행 승인은 인간에게 남긴다.

핵심 원칙은 다음과 같다.

- 근거 우선: 모델 답변은 원 출처와 인용 위치를 함께 제시해야 한다.
- 최소 권한: 역할, 그룹, KB, Tool을 업무상 필요한 범위로 분리한다.
- 인간 승인: CMS 쓰기는 초안 생성까지만 자동화하고 발행은 명시적 승인 흐름으로 둔다.
- 재현성: 모델, 프롬프트, KB 버전, Tool 호출, 입력 출처를 기록한다.
- 데이터 주권: 민감 자료의 저장 위치와 외부 모델 전송 여부를 사전에 결정한다.

## 단계별 도입

### 1단계: 안전한 개인 생산성

- SSO, 기본 RBAC, HTTPS, 백업을 설정한다.
- `Copy Editor`, `Headline Desk`, `Translation Desk` 프리셋을 Stylebook KB와 함께 만든다.
- 웹 검색 결과에 출처와 확인 필요 항목을 요구하는 공통 지침을 배포한다.
- 사람 검수의 샘플 평가로 승인 모델을 선정한다.

### 2단계: 팀 지식과 협업

- 기사 아카이브, 정책, 공개 연구 자료를 KB로 정비하고 기밀 KB를 분리한다.
- 데스크별 채널과 스토리별 비공개 채널을 만든다.
- 노트 기반 원고 작업과 채널 기반 피드백을 도입한다.
- 일간 브리프, 경쟁사 모니터링, 정정 점검을 Automations로 시범 운영한다.

### 3단계: 통제된 시스템 통합

- 읽기 전용 아카이브, 인물·용어, 팩트체크 Tools를 먼저 구축한다.
- CMS는 초안 생성 API만 연결하고, 발행 권한은 Open WebUI에 주지 않는다.
- 민감정보 차단 Filter, 감사 로그, Tool 승인·오류 처리 정책을 추가한다.
- 데이터 기자용 분석 서비스를 OpenAPI/MCP로 외부 분리한다.

### 4단계: 품질·비용 운영

- 대표 과제 기반 모델 평가와 변경 승인 게이트를 운영한다.
- 팀별 사용량, 비용, 처리 시간, 정정률, 출처 누락률을 관찰한다.
- 평가 결과에 따라 모델 라우팅, KB 구조, 프롬프트, Tools를 개선한다.

## 우선 개발 백로그

1. 출처 패킷 Tool: 주장마다 URL, 발행일, 인용문, 검증 상태를 구조화해 반환.
2. 사내 스타일·용어 검사 Action: 원고 변경 전 오류와 근거를 제시.
3. CMS 초안 Action: 제목, 본문, 태그, 출처 패킷을 `draft`로만 생성.
4. 정정 이력 검색 Tool: 관련 기사와 정정문을 찾아 재발 방지 체크리스트 생성.
5. 민감자료 경계 Filter: 보안 등급에 따라 외부 모델·웹 도구 호출을 차단.

## 성공 기준

- AI가 만든 모든 기사 관련 결과에 검토 가능한 근거 또는 "근거 없음" 표시가 있다.
- 기밀 KB 및 취재 자료는 권한 없는 사용자·도구·외부 공급자에게 전송되지 않는다.
- CMS 발행은 사람의 승인 없이는 일어나지 않는다.
- 대표 업무에서 모델 변경 전후 정확성, 편집 시간, 정정률을 비교할 수 있다.
- 플러그인과 통합은 소유자, 권한, 호출 로그, 장애 대응책을 가진다.
